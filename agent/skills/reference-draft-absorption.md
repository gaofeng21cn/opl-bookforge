# Reference Draft Absorption Skill

Use this skill inside `book-materialization` when the owner supplies a stronger comparable draft, prior version, sample chapter, edited chapter, or reference book passage that should improve first-draft quality.

Working policy:

- Treat reference absorption as an editorial-learning step, not as a copy source. Extract transferable craft patterns, reader-fit decisions, chapter rhythm, evidence handling, and layout expectations; do not copy protected prose, unique examples, unsupported claims, or source-specific facts unless they are independently present in the book source map.
- Run reference absorption before new visible chapter drafting or a major rewrite when the owner explicitly says a reference version is better, more natural, more complete, or more publishable.
- Produce a durable `reference-draft-absorption` ref or equivalent report that records:
  - reference artifact path or owner-supplied identifier;
  - comparison scope, including title/front matter, publication/review artifact label, page geometry, table of contents, chapter opening, dense body, case section, figure/table page, visual rhythm, and conclusion;
  - measurable deltas where available, such as page count, text extent, image count, image placement, TOC cleanliness, and visible production residue;
  - transferable strengths;
  - non-transferable or forbidden patterns;
  - book-specific adoption rules;
  - critique-to-workflow mapping when the reference is an owner/reviewer critique rather than only a prose sample;
  - chapter function contract updates;
  - concept map, core model map, or case evidence ladder updates when the reference exposes those defects;
  - chapter-task-card updates;
  - style-engine updates;
  - quality-gate updates;
  - remaining owner decisions or evidence gaps.
- Absorb strengths into BookForge-owned production surfaces, not into a private controller. Durable learnings should update the reader-style contract, style engine, chapter task cards, glossary, evidence map, chapter briefs, QC reports, or publication design profile.
- Prefer reference-derived rules that improve first-pass book prose:
  - chapters open from a lived scene, concrete institutional tension, practical reader question, or visible consequence;
  - abstract theme questions are rewritten into reader action questions, such as "where does the student practice this?" instead of only "where does this ability come from?";
  - sections move in a book-like arc rather than a report outline;
  - paragraphs carry explanation and consequence instead of only naming framework elements;
  - contrastive claims become relationship claims when possible: explain what each side contributes and how they meet, instead of relying on "X is not Y but Z" formulas;
  - chapter responsibility narrows to the chapter's main job, while adjacent-chapter material is handed off through transitions, task cards, or later chapters rather than previewed repeatedly;
  - front matter makes the artifact feel like a book or clearly labeled proof instead of a workflow bundle;
  - the table of contents is a reader-facing argument map: it includes parts, chapters, and substantive sections, while excluding review notes, production metadata, case-box labels, table labels, figure labels, empty headings, and blocker/status language;
  - Chinese chapter labels use natural punctuation and spacing, for example `第一章：标题`, `前言：标题`, and `结语：标题`, unless the owner-approved house style says otherwise;
  - process prose follows observable chains, including scene -> tension -> evidence -> correction -> consequence -> handoff, so the chapter teaches movement rather than only taxonomy;
  - examples and case boxes are embedded in the argument instead of appended as checklist items;
  - tables and figures are prepared by prose and then interpreted after placement;
  - chapter endings hand off to the next chapter with a reason to continue;
  - the table of contents shows full-book progression and target extent, not just coverage.
- When the reference is a structured review or critique, extract transferable process rules as well as wording preferences. Repetition findings should update chapter function contracts; unclear concept findings should update the concept map; weak evidence findings should update the case evidence ladder; weak modelization findings should update whole-book core model rules; publication-quality findings should update the publication design profile.
- Preserve the declared primary-reader priority. A reference that speaks to secondary readers may only supply compatible accessibility patterns; it must not shift the manuscript away from the primary reader promise.
- For nonfiction book-length work, distinguish completion quality from coverage. A reference version can be stronger because it has full-book extent, complete late chapters, richer chapter openings, smoother transitions, stronger case treatment, or fewer production residues. Convert those differences into chapter production gates instead of treating them as a one-time polish list.
- For PDF or proof comparisons, treat front matter and TOC defects as workflow defects. If a reference version has a cleaner cover/title page, cleaner TOC, stronger artifact labeling, better chapter punctuation, or more book-like hierarchy, absorb those lessons into the PDF/export profile and quality gate, not only into prose editing.
- Treat reference compression as a risk as well as a benefit. When a reference improves rhythm by cutting text, preserve or restore the necessary "why" layer: new evidence, conceptual boundary, process conflict, case consequence, figure/table interpretation, or institutional implication. Do not let a smoother reference remove the argument step that makes a claim convincing for the declared primary readers.
- When a reference is a local slice rather than a completed whole-book version, label the adoption scope precisely. A stronger local chapter revision can update style and chapter rules, but it must not be recorded as whole-book completion or used to bypass unfinished chapters, missing cases, figure assets, or publication proof gates.
- Run a post-absorption integrity check on the edited manuscript or affected chapters:
  - no sentence fragments, broken clauses, duplicated labels, or half-generated lines remain after the merge;
  - formulaic negation counts are interpreted with context, because some contrasts carry real conceptual work;
  - chapter budgets and target extent are not silently reduced by a polishing pass;
  - case stance, evidence gaps, and primary-reader density remain consistent with the reader-style contract.
- When a reference version exposes a systemic defect in current BookForge output, update the drafting pattern before continuing chapter production. Do not only patch the visible chapter text.
- Required fail-closed conditions:
  - reference comparison is mentioned in owner handoff but no adoption report or style/chapter-task update exists;
  - current drafts still expose review metadata, blocker phrasing, target budgets, source refs, or internal status in reader-facing prose after the reference showed a cleaner book surface;
  - review notes, production notes, case/table/figure labels, empty headings, or status metadata appear as normal TOC entries after the reference showed a cleaner book TOC;
  - chapters remain outline-like, memo-like, or checklist-like while the reference demonstrates a book-prose alternative;
  - reference absorption makes the prose shorter or smoother but removes necessary explanation, evidence, case conflict, or transition logic without recording a deliberate owner/editor decision;
  - a local reference slice is described as a whole-book completion, publication proof, or final export;
  - merged reference text introduces broken sentences, dangling fragments, duplicated numbering, or stale partial-generation residue;
  - the reference's secondary-reader accommodation silently changes the declared primary readers;
  - BookForge copies reference prose without source authority or owner permission.

This skill improves BookForge's drafting pattern. It does not authorize final editorial acceptance, publication proof readiness, or owner approval.
