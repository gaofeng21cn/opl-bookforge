# Book Authority Boundary

OPL BookForge owns book-domain semantics: storyline, chapter logic, evidence placement, style consistency, editorial quality gates, illustration and table planning, layout review criteria, and owner handoff refs.

OPL Framework owns generated interfaces, stage runtime, queue, attempt ledger, provider lifecycle, Agent Lab, projection, and framework receipts.

Book truth, manuscript body, memory body, illustration/table body, quality verdict, export verdict, and publication decision stay under OPL BookForge or the human owner gate. Scaffold validation, generated interface readiness, provider completion, or Agent Lab pass cannot claim book readiness.
