# Book Authority Boundary

OPL Book Forge owns book-domain semantics: storyline, chapter logic, evidence placement, style consistency, editorial quality gates, illustration and table planning, layout review criteria, and owner handoff refs.

Reader-facing style is book-domain truth. Book Forge may infer a reader-style contract only when the owner brief and source refs support a high-confidence conclusion; otherwise it must ask the owner or return a typed blocker before body drafting. OPL-generated surfaces cannot silently choose a generic audience or prose style.

OPL Framework owns generated interfaces, stage runtime, queue, attempt ledger, provider lifecycle, Agent Lab, projection, and framework receipts.

Book truth, manuscript body, memory body, illustration/table body, quality verdict, export verdict, and publication decision stay under OPL Book Forge or the human owner gate. Scaffold validation, generated interface readiness, provider completion, or Agent Lab pass cannot claim book readiness.
