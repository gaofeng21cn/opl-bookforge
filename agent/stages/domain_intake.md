# OPL Book Forge Domain Intake Stage

Purpose: capture the first domain-specific request, source refs, authority boundary, and handoff criteria before any OPL-hosted execution starts.

Required inputs: user intent, source locator refs, expected deliverable class, domain authority owner, and blocked-scope list.

Required outputs: intake receipt ref, accepted next-stage ref, typed blocker ref when intent or authority is unclear, and no-forbidden-write evidence ref.
